# MDM Serving/Subscribe

| 项目属性 | 详情 |
|---------|------|
| **项目名称** | MDM Serving/Subscribe |
| **项目类型** | 🔄 Ongoing（持续运营） |
| **当前状态** | 进行中 |
| **优先级** | 🔴 High |

## 项目背景

持续的数据订阅和分发需求，支持下游系统对主数据的实时或定期获取。包括整表的订阅或字段级别的数据订阅服务。

**痛点：**
- 下游系统对主数据有持续的订阅需求
- 不同应用需要不同粒度的数据（整表/字段）
- 数据更新频率要求不同（实时/批量）
- 需要灵活的订阅管理机制

## 项目目标

**主要目标：**
- 提供稳定的主数据订阅服务
- 支持灵活的订阅配置（整表/字段级别）
- 保障数据分发的及时性和准确性
- 持续优化订阅性能

## 核心功能

### 1. 整表订阅
- 完整的主数据表订阅
- 定期全量刷新
- 增量更新支持

### 2. 字段级订阅
- 自定义字段选择
- 按需订阅配置
- 字段解释和说明

### 3. 订阅管理
- 订阅申请和审批
- 订阅配置管理
- 订阅使用监控

## 技术栈

| 技术 | 用途 | 备注 |
|------|------|------|
| **Event Hub** | 事件流处理 | 实时数据分发 |
| **Azure Data Factory** | 批量数据传输 | 定期全量更新 |
| **SQL Server** | 数据存储 | 订阅配置管理 |
| **REST API** | 订阅接口 | 数据访问服务 |
