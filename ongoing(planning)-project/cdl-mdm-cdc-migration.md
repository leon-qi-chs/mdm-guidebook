## 项目背景

CDC (Change Data Capture) 功能在上线时使用了原有的 DBR (Databricks) 集群。原 DBR 集群配置特别高，导致云成本显著增加，预计会使 budget 超过预算。

**痛点：**
- ⚠️ 原 DBR 集群配置过高（高性能配置）
- 💰 云成本超出预算（FinOps 风险）
- 🔧 CDC 工作负载与原集群负载耦合
- 📊 资源利用率不合理

## 项目目标

**主要目标：**
- 将 CDC Repository 迁移到独立的 Databricks Cluster
- 优化集群配置，降低云成本
- 保持 CDC 功能稳定性和性能
- 解耦 CDC 工作负载与原有集群

**预期收益：**
- 💰 降低云成本 30-50%（通过合理配置集群）
- 🎯 提高资源利用率
- 🔒 提升系统稳定性（工作负载隔离）
- 📈 便于独立扩展和优化

### 集群配置优化

**推荐配置：**
- **节点类型**: Standard_DS3_v2 (适中性能)
- **最小节点数**: 2
- **最大节点数**: 4 (自动扩展)
- **Auto-termination**: 30 分钟（降低空闲成本）
- **Spot Instances**: 考虑使用 Spot VM（非关键时段）

**成本优化策略：**
- 使用 Databricks Auto-scaling
- 配置合理的 Cluster 自动终止时间
- 使用 Job Cluster 而非 All-purpose Cluster
- 监控和告警配置
